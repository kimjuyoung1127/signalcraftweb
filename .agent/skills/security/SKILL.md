# 보안 검토 스킬 (Security Audit Skill)

이 스킬은 `DogCoach` 프로젝트의 보안 수준을 높이고 민감한 정보가 저장소에 커밋되는 것을 방지하는 데 집중합니다.

## 주요 검토 대상
1.  **민감 정보 노출**: 파일 내에 하드코딩된 비밀번호, API 키, 인증 토큰이 있는지 확인합니다.
2.  **.gitignore 준수**: `.env`, `Databasepw.md` 등 보안상 제외되어야 할 파일이 적절히 설정되었는지 확인합니다.
3.  **권한 설정**: 민감한 정보를 다루는 파일이나 디렉토리의 접근 권한 설정을 검토합니다.

## 보안 체크리스트
- 소스 코드 내에 "password", "secret", "key" 등의 키워드가 포함된 문자열이 하드코딩되어 있는가?
- `.env` 파일이 버전 관리 시스템(Git)에 포함되어 있지는 않은가?
- 공개된 문서(`docs/`) 내에 시스템 내부 구조나 비밀번호 정보가 포함되어 있는가?
- 외부 라이브러리(dependencies) 중 보안 취약점이 보고된 버전이 있는가?

## 실행 가이드
1.  사용자가 "보안 점검해줘" 또는 "보안 문제 확인해줘"라고 요청하면 이 지침을 따릅니다.
2.  프로젝트의 모든 파일을 스캔하여 위 체크리스트 항목을 대조합니다.
3.  발견된 취약점은 위험도(High/Medium/Low)와 함께 구체적인 수정 방법을 제안합니다.
