# 코드 리뷰 스킬 (Code Review Skill)

코드를 리뷰할 때 다음 단계를 준수하여 피드백을 제공하세요:

## 리뷰 체크리스트
1. **정확성(Correctness)**: 코드가 의도한 대로 동작하는가?
2. **에지 케이스(Edge cases)**: 오류 조건이나 예외 상황이 처리되었는가?
3. **스타일(Style)**: 프로젝트의 컨벤션을 따르고 있는가?
4. **성능(Performance)**: 명백한 비효율성이나 성능 저하 요인이 있는가?

## 피드백 가이드라인
- 변경이 필요한 부분에 대해 **구체적으로** 기술하세요.
- 단순히 무엇을 바꿔야 하는지가 아니라 **이유(Why)**를 설명하세요.
- 가능한 경우 **대안**을 제시하세요.

## 실행 가이드
1.  사용자가 "코드리뷰 해줘"라고 요청하면 이 지침을 따릅니다.
